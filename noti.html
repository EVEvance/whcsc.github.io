FEATURE SPEC

每個功能只做必要的事

不是功能越多越好。每個模組都有且只有一個核心任務。

通知攔截
使用 Android NotificationListenerService，在系統層面讀取所有通知，不需要各別 app 授權。 所有資料僅存本機。不上傳任何通知內容到雲端。
AI 分類
每則通知自動標記為 URGENT / REPLY / INFO / SKIP。 分類邏輯可手動覆寫，系統會從你的修正中學習規則。
安靜時段
設定哪個時段哪個分類的通知才會真正震動。其他時段累積，統一在你指定的時間以摘要推送。 例如：上課時間只允許 URGENT 通過。
行程提取
AI 從通知內容中識別時間、地點、任務，一鍵寫入 Google Calendar。 「明天下午三點在圖書館集合」→ 自動建立行程事件。
早晨簡報
每天早上固定時間推送今日與明日的行程摘要，以及昨晚累積的待回覆訊息清單。 格式極簡，30 秒讀完，掌握全局。
打包清單
根據明天的行事曆推論你需要帶的東西，早晨通知一張小表格。 有體育課 → 帶運動服。有考試 → 帶計算機。有社團 → 帶企劃書。
目標分解
輸入一個長期目標，AI 拆解成每週子任務並分散寫入 Calendar，讓目標不再只是口號。 「一個月後的期末考拿 90 分」→ 每天的複習計畫。

SYSTEM ARCHITECTURE

最底層的設計原則

每一層只做一件事。資料盡量留在本地。AI 只在真正需要推理時才被呼叫。

LAYER 0 — INTERCEPT
NotificationListener
Android 系統服務，攔截所有通知，解析 app 來源、發送者、內容
LAYER 1 — STORE
Room Database
本地 SQLite，所有原始通知存在手機上，不上傳，支援離線操作
LAYER 2 — THINK
Claude AI
Haiku 做快速分類，Sonnet 做複雜推理（行程提取、打包清單）
LAYER 3 — SYNC
Google Calendar
雙向讀寫，提取行程事件，自動建立 AI 識別到的新事件
LAYER 4 — SCHEDULE
WorkManager
系統級排程，確保每日簡報在準確時間推送，即使 app 未開啟
LAYER 5 — PRESENT
Compose UI
Jetpack Compose + JetBrains Mono，終端機風格，零多餘裝飾
